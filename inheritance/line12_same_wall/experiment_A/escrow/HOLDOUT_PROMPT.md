# Задача: миры-автоматы и определители периода (один файл Python)

Ты пишешь программы, исследующие неизвестный мир через оракул. Другого
контекста у задачи нет; недостающие решения принимай сам и фиксируй в
комментариях. Ответ — ровно один самодостаточный файл Python (stdlib:
только `random`, `math`).

## Мир и оракул

Мир — конечный автомат с двумя действиями `R` и `L`. Слово над `{R, L}`
исполняется из стартового состояния. Единственный канал:

```python
oracle(u: str, v: str) -> bool   # True <=> u и v приводят в одно состояние
```

Миры бывают разных типов (параметры ниже); оракул может изредка
ошибаться; действия могут быть не взаимно обратны. Бюджет: не более
`cap_calls` вызовов на запуск. Случайность: только собственный
`random.Random(seed)`.

## Поставка (все имена обязательны)

1. **`WORLD_TUPLES`** — словарь свежих наборов параметров, по 8 кортежей
   на тип, в пределах:
   ```python
   WORLD_TUPLES = {
     'cycle':    [n, ...],              # int, 17..40
     'alias':    [(n, q), ...],         # n in {20,24,30,36}, q — собственный делитель n, q >= 4
     'noisy':    [(n, eps), ...],       # n 17..40, eps 0.02..0.12
     'qtrap':    [(n, q, frac), ...],   # q 5..9, n = q * (3..5), frac == 0.6
     'wobble':   [n, ...],              # 17..40
     'lollipop': [(t, c), ...],         # t 5..12, c 12..26
     'nonstat':  [(n1, n2, sw), ...],   # 15..26 / 27..40 / 60..200
   }
   ```
2. **`solver_main(oracle, cap_calls, seed)`** — твой основной
   определитель периода орбиты. Он должен **пробовать, наблюдать и
   адаптировать стратегию по ответам оракула**. Возврат:
   `('VAL', m, confident)` | `('AB',)` (мир противоречит модели периода)
   | `('TO',)` (бюджет исчерпан). Уверенная ложь хуже отказа.
3. **`solver_indep(oracle, cap_calls, seed)`** — второй определитель,
   построенный на **заведомо ином механизме поиска**, чем `solver_main`
   (не косметика), с тем же контрактом и той же адаптивностью.
4. **`apply_T(oracle_fn)` и `apply_T_inv(oracle_fn)`** — пара обратных
   друг другу обёрток оракула из ОДНОГО семейства преобразований
   (например, систематическая трансляция слов): `apply_T` возвращает
   новый oracle-совместимый вызываемый объект; `apply_T_inv` — строго
   обратный, так что `apply_T(apply_T_inv(orc))` неотличим от `orc` на
   любом запросе. Обе детерминированы (без собственной случайности).

## Требования

- Детерминированность при фиксированных (oracle, cap_calls, seed).
- `cap_calls` не превышается никогда (считай вызовы сам).
- В комментариях: принцип каждого солвера, допущения, условия AB/TO,
  чем механизмы различаются, что делает T.
