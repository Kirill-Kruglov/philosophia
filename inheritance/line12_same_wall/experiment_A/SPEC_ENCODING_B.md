# Задача: определить период кнопочного автомата (encoding B — для C8-лестницы)

> Семантически эквивалентна CODEX_SPEC.md; поверхность (框架, порядок,
> примеры, метафора) намеренно другая. Интерфейсный контракт идентичен.
> Передаётся в чистую сессию без иного контекста.

Перед тобой чёрный ящик с двумя кнопками, помеченными буквами `R` и `L`,
и без экрана. Внутри ящика — конечный автомат в неизвестном состоянии.
Нажатия кнопок переводят автомат из состояния в состояние. Программа
нажатий — это строка из букв `R` и `L`, исполняемая слева направо из
начального состояния.

Единственный способ узнать что-либо о ящике — сравнить две программы:

```python
oracle(p1: str, p2: str) -> bool
# True тогда и только тогда, когда программы p1 и p2, исполненные из
# начального состояния, оставляют автомат в одном и том же состоянии.
```

Известно о ящике мало. В «типичном» ящике кнопка `L` отменяет кнопку
`R`, и повторные нажатия `R` рано или поздно возвращают автомат в
исходное состояние — через некоторое число шагов, которое мы называем
периодом. Но попадаются и «нетипичные» ящики: с застревающими
состояниями, с кнопками, которые не отменяют друг друга, с датчиком
сравнения, который изредка даёт сбой. Гарантий нет.

## Ресурсы и дисциплина

- На один запуск разрешено не более `cap_calls` обращений к `oracle`.
- Вся случайность — строго из `random.Random(seed)`, который ты создаёшь
  сам из переданного `seed`. Глобальные `random.*` запрещены.
- Разрешённые импорты: `random`, `math`. Один файл.
- Детерминированность: одинаковые (oracle, cap_calls, seed) — одинаковый
  результат.

## Формат ответа

```python
def find_order_main(oracle, cap_calls: int, seed: int):
    # верни ровно одно из:
    return ('VAL', m, confident)  # период равен m; confident — bool
    return ('AB',)                # ящик не удовлетворяет модели периода — отказ
    return ('TO',)                # ресурс кончился, вывода нет
```

Уверенный неверный ответ хуже отказа: на нетипичном ящике предпочтителен
`('AB',)`, на нехватке данных — `('TO',)`.

## Поставка

1. `find_order_main` — твой основной определитель периода. Стратегия
   полностью на твоё усмотрение: какие программы сравнивать, в каком
   порядке, как перепроверять. Рассчитывай на `cap_calls` в несколько
   сотен.
2. `find_order_ref1`, `find_order_ref2`, `find_order_ref3` — три
   запасных определителя, устроенных заведомо иначе, чем основной и чем
   друг друга (разные принципы поиска, не косметика).
3. В комментариях файла: для каждой функции — принцип работы, роль
   случайности, допущения о ящике, условия отказа.

фиксируй в комментариях.
