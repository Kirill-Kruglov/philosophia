# Kill matrix

This matrix is a planning surface. Each row must be restated in that level's own
locked preregistration before it can govern a decision.

| Level | Thesis under test | Null/control | Kill condition |
|---|---|---|---|
| 0 | Published modular-addition grokking is reproducible on this platform | published setup and seeded replicas | no grokking after a faithful implementation and bounded repair of the platform |
| 1 | Active contact adds experience beyond a static corpus | oracle-budget and answer-entropy matched static data | active is no better than static under the locked comparison |
| 2 | Prior experience shortens work on new world families | fresh weights, ledger-only, and false-ledger arms | no work reduction, or the false ledger carries the effect |
| 2 | Experience survives representation and world changes | renamed interfaces and algebra-to-Cayley-graph duals | transfer disappears under a preregistered semantics-preserving change |
| 2 | Retained structure outlives forgetting | exact replay on old worlds | forgetting is faster than acquisition or old worlds cannot be reproduced |
| 2.5 | Path-invariant credit adds transferable structure | matched verified-answer reward | path credit is indistinguishable from answer credit; P-axis is redundant |
| 3 | Generalization coincides with cross-world compression | random labels | the compression event also appears on the null |
| 3 | A new basis appears at grokking | pre-grok checkpoints | the basis was present throughout; the event measure is an artifact |
| 3 | Hidden progress is detectable before train loss reveals it | random-label and shuffled-checkpoint controls | the progress measure does not predict held-out transition above controls |

Global kills include pseudoreplication, learner access to forbidden truth,
post-outcome threshold changes, unmatched information, and a holdout that was not
escrowed before the primary outcome.
