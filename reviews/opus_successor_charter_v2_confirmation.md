## 2. Answers to the five questions

**1. Q integrity — CONFIRMED closed on all four paths.** The six-step ordering
(§2), the conditional-independence property `W_{Q,j} ⫫ (candidate_j, all T/Q
history before attempt j) | precommitted frame/sampler/strata/attempt id` (§2),
canonical candidate identity (§4), the invariant common competence contract (§4),
the family false-pass budget `Σ α_j ≤ δ_Q` over every launchable attempt (§4),
and unified launch charging (§4) together close:
- **Prediction** — Q randomness comes into existence only *after* the manifest and
  attempt-claim are durable (steps 2→3), and candidate code receives only the
  sealed contact interface, never the seed/generator/identities/truth (step 4). A
  candidate frozen before its worlds exist cannot encode them.
- **Candidate-reset** — canonical equivalence: behavior-inert edits never reset a
  cap; any behavior-relevant change is a *new* candidate drawing the *common total
  cap*, so trivial edits and candidate re-spawning cannot buy fresh budget.
- **Crash-farming** — unified charging: every launch, *including*
  environment/resource/process/hash/seal-invalid launches, consumes its id, cap
  slot, and error allocation; crash-and-retry burns the same finite budget. Met
  **a fortiori** vs my two-ceiling design.
- **Repeated-search** — the false-pass inequality is conditional on all adaptive
  prior history, so adaptive candidate generation is explicitly covered:
  `Pr(any Q-incompetent candidate ever passes) ≤ δ_Q` regardless of search. The
  residual freedom to choose *which competent* candidate is submitted first is not
  a leak — it is priced by §1c's selection-conditional estimand.

**2. Freeze integrity — CONFIRMED.** §5 freezes the full manifest **including the
device/software stack** durable *before* that attempt's Q randomness exists,
carries it **unchanged** through automatic promotion ("it records, it does not
choose again"), makes a changed stack a new candidate, and voids the lock on any
post-lock stack change; the standalone promotion-time stack-freeze step is
removed. The **sole** active C order is unambiguous and appears once (§9):
successor scientific spec → X/Y review → author signatures → **durable
preregistration lock → post-lock secret C root generation + C realization →
ciphertext/salted-hash commitment (sealed) → separate one-shot execution
authorization → one-shot C execution → authorized unsealing**. I confirmed no
escrow-before-lock phrasing survives anywhere (the only occurrence is the §9 note
recording the correction); §2/§3 agree (root only after the complete lock).

**3. Claim integrity — CONFIRMED.** §1c makes **every** confirmatory estimand and
verdict conditional on `H_preC`, `d*`, `s*`, `L*`, averaging only over the worlds
and seed law named by `P_C`/`L*`, with a `selection_scope_id` hashing those
objects on every verdict record. The permitted-claim blockquote covers positive
and null ("confirms or fails to confirm … for this selected design and C
population only"); boundary, `INSUFFICIENT`, and the censoring terminal are C-
execution terminals (§8) under the same scope; and `PROOF_CORE`/`PROOF_STRONG`, if
earned, are "design-conditional in the estimand itself." The forbidden list
explicitly bars validating the candidate generator, the search/promotion
procedure, the learner class, Route B generally, or any unselected design. So no
statement of any type escapes the selected-design/locked-population scope.

**4. No-scout default — CONFIRMED complete and honest, no hidden direction
channel.** Under the default (§6): sample size from a **precommitted conservative
rule** (bounded-outcome/Popoviciu variance on the locked endpoint scale + the
signed resource envelope — a fixed formula, "conservative means possibly larger C,
never data-tuned smaller"); engineering caps from **label-free Q telemetry**,
sound precisely because **Q has no arms** — Q is a single-candidate competence
gate, so its resource aggregates carry no contrast/direction; and endpoint,
budgets, margins, treatment, population, and analysis are "never planning-data-
informed by any surface." The only dev→spec inputs are a fixed conservative rule
and arm-free engineering aggregates, so there is no hidden direction channel. The
one residual (possibly-large C) is a priced cost, not a leak, with strict-S as a
signed-amendment escape.

**5. Strict-S negative space — HARMLESS for the no-S tokens; one sentence
recommended, not a blocker.** §9 places optional S before the "successor
scientific specification," while §6 requires the endpoint horizon, treatments,
contrast family, margins, and analysis frozen *before* S runs — so on the S path
those scientific quantities must freeze in a pre-S skeleton lock that §9 does not
diagram as a distinct step. This is **harmless because**: (a) under the default
there is **no S**, the requirement never triggers, and §9's order is unambiguous
(promotion → full spec → lock); (b) adopting S requires a **separate signed
amendment that amends the architecture token** (§6/§11) and is itself X/Y-reviewed,
and (c) §6 is **normative** and already requires the pre-S freeze, so §9's diagram
is merely less explicit, not contradictory. Therefore **v2 need not fully name a
pre-S skeleton lock now** (that would over-engineer an unauthorized path); it
should add one clarifying sentence assigning the pre-S-lock obligation to the S-
authorizing amendment. **This does not block `I_ACCEPT_PHILOSOPHIA2_CHARTER_V2`,
`I_ACCEPT_THREE_SURFACE_PHASE_ARCHITECTURE`, or
`I_ACCEPT_SELECTION_CONDITIONAL_CONFIRMATORY_CLAIM`,** all of which accept the no-S
default.
