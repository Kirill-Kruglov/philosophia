## 2. Decisive argument for the three-surface architecture

Two surfaces cannot deliver Route B's mandate. With only
development + confirmation, learner competence is established either
(a) inside the confirmatory contract — the stopped line's one-shot
architecture, which the author decision explicitly replaces — or
(b) on the tuning registry itself, which open iteration may have
overfit, so "competence" would be indistinguishable from registry
memorization. The qualification surface is the minimal repair: it is
the *only* place where competence is demonstrated on worlds the
candidate has never touched, before any scientific contract exists,
under a predicate fixed before results are inspected. Its three
defenses are structural, not promissory: disjoint fresh worlds per
attempt (defeats registry overfitting), a frozen candidate manifest
(defeats silent tuning-through-the-gate), and capped logged attempts
(defeats qualification-by-exhaustion). Secrecy is used only where it is
load-bearing (C worlds, post-lock); everywhere else the protection is
disjointness plus pre-commitment, which — unlike secrecy — is auditable
after the fact.

### 5a. Ten mandated risks — prevention or residual

1. Dev overfitting as competence → Q disjoint fresh worlds per attempt;
   promotion requires Q, never T performance. *Mechanical.*
2. Qualification as citable pilot → binary-only surface, no series,
   permanent `dev-non-citable` labels. *Mechanical.*
3. Confirmatory worlds indirectly selected → C partition signed before
   any T generation; C realized post-lock from escrow-secret root
   nobody inspects pre-lock. *Mechanical.*
4. Outcome-triggered device choice → stack frozen at promotion, before
   lock; post-lock change voids the lock. *Mechanical.*
5. Indefinite development → declared envelope, author stop, capped
   attempts; extension only by loud signed decision. *Mechanical.*
6. Undeclared multiplicity → qualified set closes at promotion; exactly
   one candidate, one confirmatory run. *Mechanical.*
7. Old-artifact leakage → quarantine table; no warm starts; old entropy
   never reused. *Mechanical.*
8. Estimand drift via simultaneous learner+world change → construct
   fixed at charter; world-class change = loud charter amendment; C1
   skeleton and endpoint class fixed. *Mechanical at class level;
   residual: within-class drift is judged at spec review.*
9. Gate narrated as wall → `I_RESERVE_WALL…` token governs prose;
   validity-first taxonomy governs records. *Mechanical for records;
   prose needs editorial vigilance — residual.*
10. Successor as positive-ending-seeking → "no qualifier" is a named
    publishable ending; negative/`INSUFFICIENT` destinations pre-signed
    at spec time; the confirmatory claim is explicitly
    selection-conditional, so a null is a clean publishable
    confirmatory null. *Structural; residual: motivation cannot be
    mechanized, only priced.*
